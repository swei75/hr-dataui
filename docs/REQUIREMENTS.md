## 1. 项目背景

本项目是一个**人力资源管理数据驾驶舱**生成工具。

**业务输入**：用户维护一份 Excel 模板（9 个 sheet：1 配置 + 6 模块 + 1 多列 + 1 _prev 快照），每月更新数据后运行 Python 构建脚本。

**业务输出**：单个 `index.html` 文件（v1.5：~98KB），包含当月所有数据的多模块可视化仪表盘 + Dashboard 顶部数字塔。

**业务场景**：银行 HR 月度报表（2026 年 5 月起），需要分发给领导、HR 业务部门，以及作为历史档案留存。

### 3.7 v1.5.21.3 补丁：移动端响应式

桌面端 ≥1100px **完全不动**（用户原话"不要破坏桌面端浏览器下显示效果"）；<1100px viewport 触发响应式：

**3 个断点**：1100 / 768 / 480px（与 `templates/base.py` @media 一致）

**核心模式**：17 个内部 grid 改为 `repeat(N, minmax(MINpx, 1fr))` + `overflow-x: auto`

| Grid 类 | 桌面列数 | MIN | 移动行为 |
|---|---|---|---|
| `.kpi-hero` | 6 | 180 | 6×180=1080 > 容器 → 6 列横滚 |
| `.kpi-cards-row` (M-1) | 6 | 160 | 6×160=960 → 6 列横滚 |
| `.kpi-row.v11` (M-2 KPI) | 4 | 200 | 4×200=800 → 4 列横滚 |
| `.dim-grid` (M-2 stacked) | 5 | 160 | 5×160=800 → 5 列横滚 |
| `.cs-grid` (M-2 8 卡) | 4 | 200 | 4×200 → 4 列横滚 |
| `.fc-grid` (M-5) | 5 | 200 | 5×200 → 5 列横滚 |
| `.card-wall-m6` (M-6) | 6 | 160 | 6×160 → 6 列横滚 |
| ...（其余 10 grid 同模式）| — | 150-300 | — |

**不**强制单列堆叠（v1.5.21.3 初版的"一个接一个 1fr"已被初版用户否定）

**移动 scrollbar 可见**（`templates/base.py` 1100 段）：
- `::-webkit-scrollbar { height:6px; width:6px }`
- `::-webkit-scrollbar-thumb { background: var(--primary) }` 红棕色
- Android Chrome 生效；iOS Safari 系统强制隐藏不可控

**viz/*.py 兜底**：
- `viz/bar.py` `min-width:120px` 和 `viz/funnel.py` `min-width:120px` 通过 `.bar-label/.funnel-row{min-width:0}` override
- `viz/donut.py` SVG `120×120` → CSS `.donut-svg{80×80}` 缩减

**保持不变**：v1.5.21 顶部 hero 广银红+金；6 模块（M-1~M-6）主调色板；所有 v1.5.21.2 5 处排版 bug 修复

**回滚**：

```bash
cp backup/pre-mobile-2026-07-05/base.py templates/base.py && python build.py
```

## 8. 数据契约（v1.5：10 列）

### 8.1 Excel 结构

```
data/
└── *.xlsx
    ├── 配置            # 报告标题、报告期、机构名、主色
    ├── M-1 组织架构
    ├── M-2 员工情况
    ├── M-3 人员优化
    ├── M-4 干部队伍
    ├── M-4 干部职数表  # 多列结构
    ├── M-5 考核薪酬
    ├── M-6 培训赋能
    └── _prev           # 快照（composer 自动维护）
```

### 8.2 10 列数据契约

每个数据 sheet **10 列固定结构**：

| # | 列 | 类型 | 决定什么 |
|---|---|---|---|
| 1 | 分组 | string | 数据分类（影响 viz 选择） |
| 2 | 名称 | string | 显示名 |
| 3 | 数值 | number | 主要数字 |
| 4 | 单位 | string | 显示单位 |
| 5 | 备注 | string | 可选说明 |
| 6 | 排序 | int | 显示顺序 |
| 7 | is_total | bool | 是否总数行 |
| 8 | delta | auto | 环比差（composer 自动算） |
| 9 | sub_text | string | 模块级叙述（关键洞察） |
| 10 | metric_note | string | KPI 卡底部说明（12-16 字看图说话） |

### 8.3 约定

- 占比统一 0-1 浮点（`0.2586` = 25.86%）
- 缺失值 = 空 cell（**不写** "N/A" / "无" / "-"）
- 千分位统一 `,`（`5,901`）
- 时段字段从 `配置` sheet 读，不写死

## 12. 决策记录（Decisions）

v1.5 关键决策：

| # | 决策项 | 选定方案 | 理由 |
|---|---|---|---|
| D-1 | 图表库 | **CSS 静态图表（每模块独立 viz）** | 参考 shanjinki，v1.5 多 viz 函数提升表达力 |
| D-2 | 样式库 | **自研 CSS（参考 shanjinki 18 styles）** | 不引入 Pico.css，保持单文件最小体积 |

| D-4 | viz 风格 | **每模块独立风格（v1.5 革新）** | 比统一默认风格表达力更强 |
| D-5 | Excel 表头 | **中文表头 + mapping.py** | 用户友好、字段映射集中可维护 |
| D-6 | 时段字段 | **Excel 中可配置** | 各月报告期不同 |
| D-7 | 数据契约 | **10 列**（加 delta + sub_text + metric_note） | ADR-011 调研结论 |
| D-8 | 参考项目 | **fork shanjinki/excel-to-html-slides** | MIT 协议 |
| D-9 | 主色 | **#9f6b44 米色红棕** | v1.5 调研 + 用户确认 |
| D-10 | 字体大小 | **body 16px / hero 3.4em / module-title 2em / kpi-card 2.4em** | 加大字号，提升专业感 |

## 13. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| Excel 模板结构频繁变动 | 抽取器需频繁更新 | 字段映射集中到 `mapping.py`，变更可控 |
