# hr-dataui 项目需求文档

> **版本**：v1.0
> **更新日期**：2026-07-03
> **状态**：✅ 已批准

---

## 1. 项目背景

本项目是一个**人力资源管理数据驾驶舱**生成工具。

**业务输入**：用户维护一份 Excel 模板（6 个工作簿，分别对应 6 大业务领域），每月更新数据后运行 Python 构建脚本。

**业务输出**：单个 `index.html` 文件，包含当月所有数据的多模块可视化仪表盘。

**业务场景**：银行 HR 月度报表（2026 年 5 月），需要分发给领导、HR 业务部门，以及作为历史档案留存。

---

## 2. 核心目标（按重要性排序）

| # | 目标 | 说明 |
|---|---|---|
| G-1 | **零部署** | 单个 `index.html` 文件，可邮件发送、U 盘拷贝、内网共享、扔进 GitHub Pages |
| G-2 | **跨平台** | PC、平板、手机三端响应式适配，无需额外配置 |
| G-3 | **数据灵活展示** | 同一份数据支持多种视图切换（按维度、按图表类型） |
| G-4 | **现代美观** | 参考当前主流 SaaS 仪表盘的视觉语言 |
| G-5 | **Excel 模板化** | 用户可独立维护 Excel，新月份可快速生成新报告 |

---

## 3. 非目标（明确不做）

为了避免范围蔓延，以下功能**明确不在 v1.0 范围内**：

- ❌ 用户登录 / 权限管理
- ❌ 后端服务（不接 API、不连数据库）
- ❌ 实时数据刷新（数据是月度快照）
- ❌ 多人协作 / 数据编辑
- ❌ 数据修改（只读展示）
- ❌ 导出 PDF / Excel（用浏览器原生打印）
- ❌ 多语言（仅中文）
- ❌ 移动端原生 App
- ❌ 自定义主题（颜色、字体）
- ❌ 暗色模式（v1.0 暂不做）

## 5. 功能需求

### 7.2 统一数据契约

构建后生成的中间数据结构（Python dict，最终 inline 进 HTML）：

```python
{
    "metadata": {
        "title": "人力资源管理数据驾驶舱",
        "period": "2026年5月末",
        "generated_at": "2026-07-03T10:00:00Z"
    },
    "modules": {
        "organization": { ... },   # 模块1
        "employees":     { ... },   # 模块2
        "optimization":  { ... },   # 模块3
        "cadres":        { ... },   # 模块4
        "compensation":  { ... },   # 模块5
        "training":      { ... }    # 模块6
    }
}
```

每个模块内部结构由各 sheet 决定，**由 `pandas.read_excel(sheet_name=None)` 自动解析为嵌套 dict**。

### 7.3 字段命名约定

- 中文表头 → 自动转为拼音键名（如 "在职人数" → `on_duty_count`）
- 字段映射规则在 `extractors/mapping.py` 集中配置
- 关键字段命名示例：
  - `total` / `count` / `amount`：数值
  - `ratio` / `percentage`：百分比（0-1 或 0-100，需文档化）
  - `name` / `code`：标识符
  - `period`：时间段

## 9. 技术架构

### 9.1 选型决策（v1.1 更新：参考 shanjinki/excel-to-html-slides）

| 层 | 选型 | 版本 | 体积 | 理由 |
|---|---|---|---|---|
| 构建脚本 | Python | 3.10+ | - | 用户熟悉 |
| 数据处理 | pandas + openpyxl | latest | - | Excel 读取标准 |
| 模板引擎 | 字符串拼接（参考 shanjinki） | - | - | 不引入 Jinja2，减少依赖；shanjinki 用 Python f-string 生成 HTML |
| 前端框架 | Vanilla JS + Alpine.js（v1.1+ 可选） | - | - | shanjinki 用 vanilla JS；Alpine.js 留给视图切换扩展 |
| 图表库 | CSS 静态图表（v1.0），ECharts 5（v1.1+ 可选升级） | - | ~5KB | 体积最小、离线强；下钻能力弱（手动实现） |
| 样式库 | 自研 CSS（参考 shanjinki 18 styles） | - | ~10KB | 不引入 Pico.css，直接继承 shanjinki 样式系统 |
| 部署 | 单文件 `index.html` | - | - | 极致可移植，0 外部依赖 |

> **参考项目**：`shanjinki/excel-to-html-slides`（MIT），2026-07-03 调研确认
> - 仓库：https://github.com/shanjinki/excel-to-html-slides
> - 调研记录：`research-demos/README.md`
>
> **v1.0 → v1.1 升级路径**（如需下钻/联动）：
> - 局部引入 ECharts 5（CDN 模式约 1MB，inline 模式约 1MB）
> - 用于复杂图表：漏斗、桑基图、地图、联动柱图
> - 不替换 KPI 卡、表格、基础柱图（CSS 保持）

> **已弃用选型**（2026-07-03 评审弃用）：
> - ~~ECharts 5~~ → 改为 v1.0 可选（v1.1+）
> - ~~Pico.css 2~~ → 改为自研 CSS（基于 shanjinki）
> - ~~Jinja2~~ → 改为字符串拼接

### 9.3 数据流

```
data/*.xlsx
    │
    │ pandas.read_excel(sheet_name=None)
    ▼
Python dict (内存)
    │
    │ extractors/modules/*.py 转换/校验
    ▼
统一 JSON dict
    │
    │ Jinja2 模板渲染 + inline JS/CSS
    ▼
HTML 字符串
    │
    │ 写入
    ▼
output/index.html (单个文件)
```

### 9.4 关键技术细节

- **CDN 引用**：默认通过 CDN 加载 Alpine / ECharts / Pico，构建时可加 `--offline` 改成 inline
- **数据 inline**：JSON 直接写入 HTML 的 `<script>` 标签，避开 `file://` 的 CORS 限制
- **样式策略**：使用 Pico 的 classless 模式 + 少量自定义 utility class（在 `<style>` 块内）
- **图表初始化**：Alpine 组件中用 `x-init` 调用 ECharts `setOption`
- **下钻实现**：ECharts 自带 `click` 事件 + Alpine 状态管理

## 12. 决策记录（Decisions）

v1.0 评审通过，以下 7 个决策点已确认：

| # | 决策项 | 选定方案 | 理由 |
|---|---|---|---|
| D-1 | 图表库 | **CSS 静态图表（v1.0），ECharts 可选（v1.1+）** | 参考项目 shanjinki 用纯 CSS 实现，体积小、离线强；ECharts 保留为 v1.1+ 的可选升级路径 |
| D-2 | 样式库 | **自研 CSS（参考 shanjinki 18 styles）** | 不引入 Pico.css，保持单文件最小体积；shanjinki 风格已覆盖 dark/light/professional |
| D-3 | 数据粒度 | **每模块独立 Excel** | 文件名直接对应模块、便于维护 |
| D-4 | 视图切换 | **v1.0 先做 M-2 / M-3 / M-6**，其余 v1.1 | 验证模式后再推广 |
| D-5 | Excel 表头 | **中文表头 + mapping.py** | 用户友好、字段映射集中可维护 |
| D-6 | 时段字段 | **Excel 中可配置** | 各月报告期不同 |
| D-7 | 历史对比 | **v1.0 不做** | 推迟到 v1.1 |
| D-8 | 参考项目 | **fork shanjinki/excel-to-html-slides** | MIT 协议，单文件 + 中文 + HR demo 现成；2026-07-03 评审通过 |

## 13. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| Excel 模板结构频繁变动 | 抽取器需频繁更新 | 字段映射集中到 `mapping.py`，变更可控 |
| 报告期数据量增长超 1000 行 | 渲染卡顿 | ECharts 支持大数据集 + canvas 渲染；必要时分页 |
| 用户在内网无 CDN 访问 | 页面打不开 | 提供 `--offline` 模式 |
| 中文表头解析歧义（重名 sheet） | 数据错位 | mapping.py 强制显式声明 key |
| IE 兼容要求（万一） | 现有方案不支持 | 已确认 IE 不在支持范围；如需，加 polyfill |
