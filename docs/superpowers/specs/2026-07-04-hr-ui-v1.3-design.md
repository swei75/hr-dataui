## 1. 背景与目标

HR-UI 是银行 HR 月度管理驾驶舱工具。

**核心目标**：
- 用户维护 1 个 `data/YYYY年M月.xlsx`（source of truth）
- 每月只改 Excel 数字 → `python build.py` → 重新生成 `output/index.html`
- 接收方双击 HTML 即用，**零技术门槛、零外部依赖、完全离线**

**版本演进**：
- v1.0 需求文档（2026-07-03 前）
- v1.1 GitHub 调研 + fork shanjinki 决策
- v1.2 mapping.py + viz 注册表 + Alpine.js 钻取
- v1.3 **Excel source of truth + 分组驱动 + 自动 viz 选择**（当前实现）

## 2. 范围

### 数据源
- **1 个 .xlsx 文件**，8 个 sheet
- 配置 sheet + 6 模块 sheet + 1 多列表（干部职数表）

### 模块
6 大模块：组织架构 / 员工情况 / 人员优化 / 干部队伍 / 考核薪酬 / 培训赋能

### 可视化（自动选择）
- 根据"分组"结构和 `is_total` 标记自动选 viz：
  - 1 行（is_total）→ **KPI 卡**
  - 2-3 行（无 is_total）→ **donut**（组成关系）
  - 4+ 行（无 is_total）→ **horizontal bar**
  - 含 is_total + sub → **KPI 总数 + bar 子项**
  - 多列 sheet → **data-table**

### 多端
- 桌面（≥ 980px）/ 平板（640-980px）/ 手机（< 640px）
- 兼容：Chrome 80+ / Edge 80+ / Firefox 70+ / Safari 13+ 桌面 + 移动

### 非目标（CLAUDE.md §6 黑名单严格执行）
- ❌ React/Vue/Tailwind/CDN
- ❌ 后端/API/登录
- ❌ 多文件 SPA

```

   ↓
```

### Excel 结构（关键）

每个模块 sheet 列：
| 分组 | 名称 | 数值 | 单位 | 备注 | 排序 | is_total |
- 相同 `分组` = 同一逻辑组
- `is_total=TRUE` = 该组总数行
- `排序` 0/1/2/3 = 视觉层级

## 4. viz 选择逻辑（自动）

```python
def _pick_viz(rows):
    total = [r for r in rows if is_total]
    sub = [r for r in rows if not is_total]

    if total and not sub:  return "kpi"           # 全是总数 → KPI
    if total and sub:      return "total_sub"      # 总数+子项 → KPI+bar
    if len(rows) == 1:     return "kpi"           # 1 项 → KPI
    if 2 <= len(rows) <= 3: return "donut"         # 2-3 项 → 圆环（组成）
    if len(rows) >= 4:     return "bar"           # 4+ 项 → 横向柱
```

**关键设计原则**：
- **组成关系用 donut**（2-3 项相加 = 总数）
- **分布对比用 bar**（4+ 项）
- **总数 + 拆分** = KPI + bar 组合
