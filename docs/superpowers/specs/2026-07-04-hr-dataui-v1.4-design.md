## 2. 数据契约（ADR-011）

### 2.1 9 列结构

| # | 列 | 类型 | 来源 | 说明 |
|---|---|---|---|---|
| 1 | 分组 | string | ADR-002 保留 | 数据分类（决定 viz 类型） |
| 2 | 名称 | string | ADR-002 保留 | 显示名 |
| 3 | 数值 | number | ADR-002 保留 | 主要数字 |
| 4 | 单位 | string | ADR-002 保留 | 显示单位 |
| 5 | 备注 | string | ADR-002 保留 | 可选说明 |
| 6 | 排序 | int | ADR-002 保留 | 显示顺序 |
| 7 | is_total | bool | ADR-002 保留 | 是否总数行 |
| 8 | **delta** | auto | **ADR-011 新** | 同比/环比（composer 自动算） |
| 9 | **sub_text** | string | **ADR-011 新** | 叙述字段（可选） |
| 10 | **metric_note** | string | **ADR-011 新** | KPI 下方看图说话（可选） |

> 注：metric_note 是单列，但按分组应用——通常填在 `is_total=True` 行。

### 2.2 `_prev` 历史快照

- **存储位置**：同 Excel 的 `配置` sheet 旁边新增 `_prev` sheet
- **创建时机**：首次 `python build.py` 时为空；每月构建后**覆盖**写入当前 sheet
- **读取时机**：构建开始时读 `_prev` → 用作 delta 计算的对照
- **保留窗口**：只保留 1 期（v1.4 YAGNI，多期对比 v2.0）

### 2.3 delta 自动计算规则

```python
# composer 伪代码
def calc_delta(current_value, prev_value):
    if prev_value is None or prev_value == 0:
        return None  # 无历史数据，不显示
    diff = current_value - prev_value
    pct = diff / abs(prev_value) if prev_value != 0 else 0
    return {"abs": diff, "pct": pct, "direction": "up" if diff > 0 else "down"}
```

**显示规则**：
- `delta` 非空 + `is_total=True` → KPI 卡右下角"vs 上月 +5.2%"（绿涨/红跌）
- `delta` 非空 + `is_total=False` → bar 末端徽章"vs 上月 +50"
- `delta` 为空 → 不显示

### 2.4 sub_text 字段（叙述）

- 位置：第 9 列，**可选**
- 典型内容：
  - 模块级："本月净增 50 人，主要来自校园招聘（占比 60%）"
  - 风险级："Q2 离职率较 Q1 上升 1.2pp，需关注 90 后群体"
- 渲染：
  - **2 栏叙述**：左侧"关键洞察" + 右侧"口径假设"（参考 shanjinki-finance）
  - **风险清单卡片**：2 列卡片，每张标题 + 详情 + 行动建议

### 2.5 metric_note 字段（看图说话）

- 位置：第 10 列，**可选**
- 长度限制：12-16 字（参考广扫描 4/8 国内 BI 标配）
- 渲染：KPI 卡下方，灰色小字

## 3. composer 修订

### 3.1 数据流

```
[Excel .xlsx]
    │
    ├── 1 个 sheet: 配置（+ _prev 子表）
    ├── 6 个 sheet: 模块（M-1 ~ M-6）
    └── 1 个 sheet: M-4 多列
    │
    ▼ openpyxl 读
[reader.read_workbook(path)]
    │ 返回 dict
    │ 同时读 _prev sheet（如果有）
    ▼
[mapping.apply(raw_data, prev_data)]
    │ 对每个 sheet
    │ if is_total: 用 prev_data 算 delta
    │ 按 MODULES 配置标准化
    ▼
[composer.compose(modules)]
    │ 对每个模块 → 按"分组"切分
    │ 每个分组 → _pick_viz(group)
    │ 自动识别完成率 → 套 3 色风险
    │ 渲染 sub_text → 叙述 / 风险清单
    │ 渲染 metric_note → KPI 下方
    ▼
[templates/base.render_page()]
    │ Python f-string 渲染
    │ inline CSS（含 3 色风险 + 3px 彩条 + 章节编号 + 8 进制 spacing）
    │ inline JSON（window.HR_DATA，含 delta）
    ▼
[output/index.html]  ~78KB
```

### 3.2 viz 自动选择扩展

v1.3 规则不变，加 1 条：

| 条件 | viz |
|---|---|
| 1 行 | KPI |
| 2-3 行 + is_total | donut |
| 4+ 行 | bar |
| 多列 | table |
| **M-2 模块 + 分组是"员工 × 司龄"** | **heatmap（5×5）** |

### 3.3 完成率识别 → 3 色风险

```python
def classify_rate(value, target=1.0):
    """识别完成率类数据，返回风险等级"""
    if value >= target:
        return "success"  # 绿
    elif value >= target * 0.8:
        return "warning"  # 橙
    else:
        return "danger"   # 红
```

**触发条件**（自动识别，无需用户配）：
- 单位包含 "%" 或 "率"
- 或备注包含 "完成率" / "达标率"

## 4. 视觉修订（ADR-012 + 3 微调）

### 4.1 3 色风险等级 token（ADR-012）

```css
/* 在 templates/base.py 现有 --primary 基础上追加 */
--success:         #16A34A;
--success-foreground: #FFFFFF;
--warning:         #D97706;
--warning-foreground: #FFFFFF;
--danger:          #DC2626;
--danger-foreground:  #FFFFFF;
```

### 4.2 KPI 顶部 3px 彩条（新建议 a）

```css
.kpi-card {
  border-top: 3px solid var(--primary);  /* 默认 indigo */
}
.kpi-card[data-category="success"] { border-top-color: var(--success); }
.kpi-card[data-category="warning"] { border-top-color: var(--warning); }
.kpi-card[data-category="danger"]  { border-top-color: var(--danger); }
```

`data-category` 由 composer 自动填（基于完成率识别或 KPI 类型）。

### 4.3 章节编号 01/02/03（新建议 a）

```css
.module::before {
  content: counter(module) " · ";  /* "01 · 组织架构" */
  counter-increment: module;
}
body { counter-reset: module; }
```

### 4.4 8 进制 spacing（新建议 a）

```css
/* v1.3 → v1.4 改动 */
--space-1: 8px;     /* v1.3: 4px  → 保留作为最小 token */
--space-2: 16px;    /* v1.3: 8px  → v1.4: 16px */
--space-3: 24px;    /* v1.3: 16px → v1.4: 24px */
--space-4: 32px;    /* v1.3: 32px → v1.4: 32px */
--space-5: 48px;    /* v1.3: 无   → v1.4: 新增 */
--space-6: 64px;    /* v1.3: 无   → v1.4: 新增 */
```

**迁移策略**：v1.3 的 `--space-1=4px` 保留作为内部最小 token，但模块级间距全部用 8 进制。

## 5. heatmap（M-2 限定）

### 5.1 触发条件

- 仅 M-2 模块启用
- 数据结构：分组 = "员工类型 × 司龄"
- 渲染：5×5 网格（5 种员工类型 × 5 段司龄：<1年 / 1-3 / 3-5 / 5-10 / >10）

### 5.2 视觉规范

```css
.heatmap {
  display: grid;
  grid-template-columns: repeat(5, 1fr);
  gap: var(--space-1);
}
.heatmap-cell {
  aspect-ratio: 1;
  display: flex;
  align-items: center;
  justify-content: center;
  font-size: 14px;
  font-weight: 500;
  border-radius: 4px;
  background: var(--success);  /* 浅 → 深：绿/橙/红 */
}
.heatmap-cell[data-level="warning"] { background: var(--warning); }
.heatmap-cell[data-level="danger"]  { background: var(--danger); }
```

### 5.3 数据契约（M-2 特殊）

M-2 sheet 第 8-10 列与其他模块相同。但 heatmap 渲染时需要"行 × 列"二维结构——由 composer 读 `分组` 字段解析（如 `员工-产品经理` × `司龄-3-5年`）。

## 6. 不可动项（重申）

| 不可动 | 理由 |
|---|---|
| C-1 单文件 HTML | 邮件/U 盘/内网分发核心场景 |
| C-2 0 外部依赖 | 接收方无技术背景 |
| C-3 完全离线 / file:// 可用 | 避开 CORS |
| C-4 多端响应式 | 领导可能在手机看 |
| C-5 体积目标 ≤ 50KB（实际 ~78KB 可接受） | 邮件附件友好 |
| ADR-001 单 Excel 数据源 | HR 月报场景已固化 |
| ADR-005 Python f-string | 零依赖 |
| ADR-010 单文件部署 | C-1 重申 |
