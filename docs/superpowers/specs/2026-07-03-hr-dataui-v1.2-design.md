## 1. 背景与目标

hr-dataui 是银行 HR 月度管理驾驶舱工具，将月度 .docx 报告转结构化数据并以单文件 HTML 仪表盘形式分发。

**核心目标**：
- 用户 HR 同事从 .docx 转出 1 个 Excel
- 维护者跑 `python build.py` → 生成单文件 HTML
- 接收方双击 HTML 即用，**零技术门槛**、**零外部依赖**、**完全离线**

**v1.2 变更**（相对 v1.1）：
- v1.1 完成了 GitHub 调研与选型评审（fork shanjinki）
- v1.2 新增：**mapping.py 配置化架构 + viz 注册表 + Alpine.js 钻取 + 多端响应式**
- v1.1 → v1.2 不改变核心选型（shanjinki fork / Python f-string / CSS 图表 / 0 CDN）

## 3. 架构

### 3.1 核心理念

**mapping 配置驱动 + viz 注册表渲染 + Alpine.js 钻取。**

```
数据 (Excel) → reader → mapping 配置 → viz 注册表 → HTML 片段 → composer → 单文件 HTML
                                                                     ↓
                                                              Alpine.js (inline)
                                                                     ↓
                                                              钻取 / viz 切换交互
```

### 3.3 数据流

```
[Excel .xlsx]
   │  openpyxl 读
   ▼
[reader.read_workbook(path)]
   │  返回 dict {"组织架构": [records], "员工情况": [...], ...}
   ▼
[extractors.mapping.apply(raw_data)]
   │  按 MODULES 配置 → 标准化数据 + viz 选型
   ▼
[composer.compose(modules)]
   │  对每个 section → viz_registry[type].render(data, options) → HTML 片段
   │  拼装进模板
   ▼
[composer.build_html()]
   │  Python f-string 渲染（不用 Jinja2）
   │  注入 Alpine.js (inline, 来自 vendor/alpine.min.js)
   ▼
[output/index.html]                       # ≤ 50KB
```

### 3.4 错误处理

| 错误 | 处理 |
|---|---|
| Excel 文件不存在 | `FileNotFoundError`，build 失败，stderr 提示路径 |
| sheet 缺失 | stderr 警告 + 该模块用空数据占位 + 退出码 1 |
| 列名不匹配 mapping | stderr 列出未识别列；mapping.py 配 fallback 字段 |
| viz 渲染抛异常 | 该 section 渲染为错误块 + stderr traceback |
| 输出 HTML > 50KB | build 退出码 1 + stderr 报警（不阻断） |
| Jinja2 / CDN 引用 | 不允许（用 grep 检查 CI） |

### 3.5 测试策略

- `test_reader.py`：6 sheet 解析、缺失值、千分位、占比 0-1
- `test_mapping.py`：字段识别率 ≥ 80%，错误定位到文件名+sheet+列
- `test_viz.py`：8 个 viz 函数 snapshot 测试，输出非空 HTML
- `test_build_e2e.py`：mock Excel → build → 验证 output/index.html 结构

## 4. 数据模型 & mapping.py 格式

### 4.1 Excel 结构

**1 个文件 7 sheet**：

| sheet 名 | 类型 | 行数预估 |
|---|---|---|
| `[配置]` | 配置 | ~10 行 |
| `组织架构` | 数据 | ~30 行 |
| `员工情况` | 数据 | ~50 行 |
| `人员优化` | 数据 | ~60 行 |
| `干部队伍` | 数据 | ~30 行 |
| `考核薪酬` | 数据 | ~20 行 |
| `培训赋能` | 数据 | ~40 行 |

**每个数据 sheet 形态**：
- 第 1 行：维度分类（如"年龄分布"/"客户经理分类"）
- 第 2 行：列名（中文）
- 第 3+ 行：数据
- 例外：组织架构是树形（用缩进表示层级）

### 4.2 数据契约（沿用 CLAUDE.md §8）

- 占比统一 0-1 浮点（如 `0.2586` 表示 25.86%）
- 缺失值 = 空 cell（**不写** "N/A" / "无" / "-"）
- 数字千分位统一 `,`（`5,901`）
- 时段字段从 `[配置]` sheet 读，不写死

### 4.3 [配置] sheet 格式

| 行 | key | 说明 | 示例 |
|---|---|---|---|
| 1 | 报告标题 | H1 | 人力资源管理数据驾驶舱 |
| 2 | 报告期 | H2 副标题 | 2026年5月 |
| 3 | 机构名 | 全行 | XX 银行 |
| 4 | 主色 | 主题色 hex | #1E5BAA |
| 5 | 副色 | 强调色 | #E07B39 |
| 6 | 字号档位 | S/M/L | M |
| 7 | 生成日期 | 自动 | 2026-07-03 |

### 4.4 mapping.py 格式

```python

        "sections": [
            {
                "key": "headcount_by_region",
                "title": "分行网点分布",
                "type": "hierarchy",
                "alt_types": ["table"],

    "员工情况": {

        "icon": "👥",
        "order": 2,

            {
                "key": "headcount_total",
                "title": "员工总数与构成",
```

### 4.5 数据形状约定

每个 section 的 `data` 字段（reader 输出）符合以下约定之一：

| data 类型 | 形状 | 例子 |
|---|---|---|
| KPI | `{value: number, sub: str, tone: str}` | `{value: 5901, sub: "正式员工", tone: "blue"}` |
| Distribution | `[{label: str, value: number, pct: float}, ...]` | `[{label: "30岁以下", value: 1526, pct: 0.2586}, ...]` |
| Ranking | `[{rank: int, name: str, value: number}, ...]` | `[{rank: 1, name: "总营", value: 295}, ...]` |
| Progress | `{label: str, current: number, target: number, pct: float}` | `{label: "腾笼换鸟", current: 201, target: 410, pct: 0.50}` |
| Hierarchy | `[{region: str, count: int, children: [...]}, ...]` | `[{region: "广州", count: 107, children: [...]}, ...]` |
| Table | `[{col1: val, col2: val, ...}, ...]` | `[{职数: 8, 实配: 8, 空缺: 0}, ...]` |

## 5. viz 注册表

### 5.1 注册表 API

```python
# viz/__init__.py
from . import kpi, bar, pie, funnel, progress, ranking, hierarchy, table

VIZ_REGISTRY = {
    "kpi": kpi.render,
    "bar": bar.render,
    "pie": pie.render,
    "funnel": funnel.render,
    "progress": progress.render,
    "ranking": ranking.render,
    "hierarchy": hierarchy.render,
    "table": table.render,
}
```

**每个 viz 函数签名**：

```python
def render(data: list | dict, options: dict) -> str:
    """返回 HTML 片段。options 包含 alt_types、drillable、theme 等"""
```

### 5.2 viz 类型清单

| viz | 适用数据 | CSS 基础 | 用途 |
|---|---|---|---|
| kpi | 单一数字 | `.kpi-card` (shanjinki) | M-5 工资总额 / M-2 总数 |
| bar | 横向柱图 | `.bar-track` + `.bar-fill` | 占比分布、排名 |
| pie | 圆环 | inline SVG | bar 的 alt viz |
| funnel | 多级流程 | `.funnel` (shanjinki) | 校招各阶段 |
| progress | 进度条 | `.progress` | 任务完成率 |
| ranking | Top/Bottom N | table + badge | 排名表 |
| hierarchy | 树形 | ul/li CSS | M-1 网点层级 |
| table | 数据表 | shanjinki table | 干部职数 |

### 5.3 viz 切换机制

```html
<div x-data="{ viz: 'bar' }">
  <div class="viz-switch">
    <button :class="{ active: viz==='bar' }" @click="viz='bar'">柱图</button>
    <button :class="{ active: viz==='pie' }" @click="viz='pie'">饼图</button>
  </div>
  <div x-show="viz==='bar'" x-html="renderBar(data)"></div>
  <div x-show="viz==='pie'" x-html="renderPie(data)"></div>
</div>
```

- 切换不重新加载数据
- 切换按钮放 section 标题右侧
- 手机端改下拉（节省横向空间）

### 5.4 钻取机制（Alpine.js）

```html
<div x-data="{ openDrill: null }">
  <button @click="openDrill='region_tree'" class="drill-btn">查看详情</button>
  <div x-show="openDrill==='region_tree'" class="drill-panel">
    <div x-html="DRILL_DATA['region_tree']"></div>
  </div>
</div>
```

- 钻取数据**预加载**到 `window.HR_DRILL_DATA`（≤ 20KB）
- 点击触发 Alpine 切换 + 渲染预加载数据
- 未来加新钻取 = mapping 标 `drillable: True` + `extractors/drills.py` 加函数

### 5.5 前端整体栈

| 资源 | 来源 | 大小 |
|---|---|---|
| HTML | 模板生成 | ~10KB |
| CSS | shanjinki 7KB + 自研 5KB | ~12KB |
| Alpine.js | inline | ~15KB |
| HR_DATA / HR_DRILL_DATA | JSON | ~10KB |
| **总计** | | **~47KB** |

目标 ≤ 50KB，超出报警。

## 8. 风险 & 缓解

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 真实 .xlsx 结构与 .docx 文本不一致 | 高 | 高 | P2 阶段先做"用 .docx 直接生成测试 Excel"脚本 |
| Alpine.js 15KB 加上后超 50KB | 中 | 中 | gzip 后实际 ~7-8KB；可考虑仅在需要钻取的模块动态加载 |
| viz 切换视觉不统一 | 中 | 低 | 所有 viz 统一从 shanjinki 调色板取色 |
| 6 sheet Excel 用户不会建 | 中 | 中 | 提供模板 `.xlsx` + 截图说明（README，可选） |
| 钻取数据量过大撑爆单文件 | 低 | 高 | 钻取数据 ≤ 20KB 上限；超出改二级加载 |
| 移动端钻取交互体验差 | 中 | 低 | 移动端钻取面板改底部 sheet |
| IE 11 兼容性 | 低 | 高 | **已确认不兼容**（用户接受） |
| Excel 旧格式 .xls | 中 | 中 | 文档强制 .xlsx |
