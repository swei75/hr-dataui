设计说明

  • 覆盖维度：12/12 完整（市管 / 高管 / 纪检 / 中层总 / 正 / 副 / 副总级一把手 / 副总 / 总助 / 经理 / 兼任 / 总行 / 经营）
  • 与 M-1 差异：M-1 是 6 个静态 KPI 全显；v10 是 4 大类折叠卡 + 点击展开（节省首屏空间）
  • 与 M-2 差异：M-2 是 5 段 stacked-bar 全显；v10 用原生 HTML <details> 元素实现可交互
  • 与 M-3 差异：M-3 v2 是 flow chart + podium（线性流程）；v10 是层级树状可展开（点哪看哪）
  • 新增 viz 类型：① 原生 HTML accordion（<details> + summary） ② 4 大类折叠卡 ③ 子分类 2 列网格 ④ 状态色边框（绿/红/中性 3 色）
  • 信息密度：高 — 4 卡 × 多子分类 × 4-5 字段 = 50+ 字段（仅展开时显示）
  • 读图速度：2-5 秒（按需展开）；首屏只看到 4 大类标题 + 3 个 quicks 数字
  • 取舍：accordion 在纯静态 HTML 中可用原生 <details> 实现零 JS 交互；默认第 1 张展开作为示例
